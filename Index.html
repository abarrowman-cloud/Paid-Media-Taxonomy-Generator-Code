Bulk Edit — Apply the Same Change Across Many Names

Paste a list of existing taxonomy names (any mix of P1 / P2 / P3), pick the field(s) you want to change, and every row gets rebuilt. Use it for common ops like changing dates, updating custom IDs, or swapping campaign names across dozens of ad names at once.

Taxonomy QA — Validate Existing Taxonomy Names

Paste P1 / P2 / P3 names (one per line) or upload a CSV / XLSX. Each name is parsed, the level is auto-detected, and the segments are checked against the spec — missing prefixes, dropdown-value mismatches, date format, funnel/objective mapping, reserved characters, and (for P3) Product SKU + Category integrity are all flagged.

UTM Builder — Append UTM Parameters

Generate a UTM-tagged destination URL. Product brands (Conair / BaBylissPRO / Cuisinart) use platform-specific fixed templates with dynamic ad-platform tokens. Unique Vacations Inc. uses a lowercase composite campaign built from 10 fields. Every generated URL is also checked for liveness — a red DEAD flag means the destination didn't respond.
★Brand

URL QA — Validate Existing UTM-Tagged URLs

Pick the brand the URLs belong to, then (for product brands) the platform. Each URL is checked against the matching template — missing keys, value mismatches, malformed dynamic tokens, cross-platform mistakes (Meta UTMs on a TikTok ad), and cross-brand mistakes (a Unique Vacations Inc. URL pasted into a Cuisinart QA) are all flagged.
Unique Vacations Inc. URLs use a single composite utm_campaign instead of platform-specific templates, so no platform pick is needed. The validator checks every utm_campaign segment against the formula: viralnation_<funnel>_<campaign>_<geo>_<objective>_<influencer>_<hotel>_<creativeType>_<assetVersion>_<adId>.
